# Step 1: Implement a Single-Server Remote Access Deployment
The first configuration step to deploy Remote Access in a load\-balanced cluster is to implement an advanced single server deployment and then plan to add servers to the cluster.  
  
## <a name="BKMK_1.1"></a>Implement a single server deployment  
Before you can configure a load\-balanced cluster, you must configure an advanced single server Remote Access deployment as described in [Deploy advanced Remote Access](assetId:///3475e527-541f-4a34-b940-18d481ac59f6).

<tr><td style='border-left-width:0px;border-right-width:0px;border-bottom-width:0px;font-size:1.5rem !important;margin-top:0px;margin-bottom:0px;' class='has-margin-top-none has-margin-bottom-none has-padding-left-none has-padding-right-none' colspan='3' width='100%'><div id='1573msgdesc'></div><h4 class='has-margin-top-none has-margin-bottom-none has-margin-right-none has-margin-left-none has-padding-left-none has-padding-right-none has-padding-top-none has-padding-bottom-none'>Unexpected results when printing from some apps or to some printers</h4></td></tr>
<tr><td width='30%' class='has-margin-top-small has-padding-top-small has-margin-bottom-small has-padding-bottom-small'><b>Status</b></td><td width='30%' class='has-margin-top-small has-padding-top-small has-margin-bottom-small has-padding-bottom-small'><b>Originating update</b></td><td class='has-margin-top-small has-padding-top-small has-margin-bottom-small has-padding-bottom-small'><b>History</b></td></tr><td class='has-no-wrap'>Resolved<a href = 'https://support.microsoft.com/help/5001648' target='_blank'> KB5001648</a></td><td class='has-no-wrap'>OS Build 18363.1440<br> <a href = 'https://support.microsoft.com/help/5000808' target = '_blank' > KB5000808</a><br>2021-03-09</td><td class='has-no-wrap'>Resolved: 2021-03-18, 19:00 PT<br>Opened: 2021-03-18, 17:25 PT</td></tr>
<tr><td width='100%' colspan='3'><div>After installing updates released March 9, 2021 or March 15, 2021, you might get unexpected results when printing from some apps. Issues might include:</div><ul><li>Elements of the document might print as solid black/color boxes or might be missing, including barcodes, QR codes, and graphics elements, such as logos.</li><li>Table lines might be missing. Other alignment or formatting issues might also be present.</li><li>Printing from some apps or to some printers might result in a blank page or label.</li></ul><div></div><div><strong>Affected platforms:</strong></div><ul><li>Client: Windows 10, version 20H2; Windows 10, version 2004; Windows 10, version 1909; Windows 10, version 1809; Windows 10 Enterprise LTSC 2019; Windows 10, version 1803; Windows 10 Enterprise LTSC 2016; Windows 10, version 1607; Windows 10 Enterprise 2015 LTSB; Windows 8.1; Windows 7 SP1</li><li>Server: Windows Server, version 20H2; Windows Server, version 2004; Windows Server, version 1909; Windows Server, version 1809; Windows Server 2019; Windows Server, version 1803; Windows Server 2016; Windows Server 2012 R2; Windows Server 2012; Windows Server 2008 R2 SP1; Windows Server 2008 SP2</li></ul><div></div><div><strong>Resolution:</strong>&nbsp;This issue was resolved in the out-of-band update&nbsp;<a href='https://support.microsoft.com/help/5001648' target='_blank'> KB5001648</a>. It is a cumulative update, so you do not need to apply any previous update before installing it. You will need to <strong>Check for updates </strong>to install <a href='https://support.microsoft.com/help/5001648' target='_blank'> KB5001648</a> and might need to also select <strong>Download and Install</strong>. For instructions on how to <strong>Check for updates</strong>, please see <a href="https://support.microsoft.com/windows/update-windows-10-3c5ae7fc-9fb6-9af1-1984-b5e0412c556a" rel="noopener noreferrer" target="_blank">Update Windows 10</a>. You can import this update&nbsp;into Windows Server Update Services (WSUS) manually. See the&nbsp;<a href="https://docs.microsoft.com/windows-server/administration/windows-server-update-services/manage/wsus-and-the-catalog-site#the-microsoft-update-catalog-site" rel="noopener noreferrer" target="_blank">Microsoft Update Catalog</a>&nbsp;for instructions.</div><div><br></div><div><strong>Note </strong><a href='https://support.microsoft.com/help/5001648' target='_blank'> KB5001648</a> is cumulative and addresses both the issue above and the issue addressed in the March 15, 2021 out-of-band release for the APC_INDEX_MISMATCH error with a blue screen.</div><div align='right'><a href ='#known-issues'>Back to top</a></div></td></tr>
</table>

<tr><td width='100%' colspan='3'><div>After installing <a href='https://support.microsoft.com/help/5000808' target='_blank'> KB5000808</a>, you might receive an APC_INDEX_MISMATCH error with a blue screen when attempting to print to certain printers in some apps. This issue affects a subset of Type 3 printer drivers and does not affect printer drivers that are Type 4. If you are unsure which type your printer driver is, use the following steps:</div><ol><li>Press the Windows key+r or select <strong>Start </strong>and type <strong>run </strong>and select it.</li><li>Type <strong>printmanagement.msc </strong>and press enter or select the <strong>OK button</strong>.</li><li>If Print Management opens, continue to step 4. If you receive the error, <strong>Windows can not find 'printmanagement.msc'.</strong> then select <strong>Start </strong>and type <strong>manage optional features</strong> and select it. Select <strong>Add a Feature</strong> and type <strong>print</strong>, select the check box next to <strong>Print Management Console</strong> and select install. You will now need to start at step 1 again.</li><li>Expand <strong>Print Servers</strong>, expand your computer's name, and select <strong>Printers</strong>.</li><li>You can now see the Driver Type for each of your installed printer drivers.</li></ol><div><br></div><div><strong>Workaround: </strong>To mitigate the issue, follow the instructions in <a href="https://www.microsoft.com/videoplayer/embed/RWzqv8" rel="noopener noreferrer" target="_blank">this video</a>. The <strong>Command Prompt </strong>commands from the video are as follows (you will need to replace <strong>KX driver for Universal printing</strong> with the name of your printer, as explained in the video):</div><ul><li><strong>rundll32 printui.dll,PrintUIEntry /Xg /n “KX driver for Universal printing”</strong></li><li><strong>rundll32 printui.dll,PrintUIEntry /Xs /n "KX driver for Universal printing" attributes +direct</strong></li></ul><div></div><div><strong>Affected platforms:</strong></div><ul><li>Client: Windows 10, version 20H2; Windows 10, version 2004; Windows 10, version 1909; Windows 10, version 1809; Windows 10 Enterprise LTSC 2019; Windows 10, version 1803</li><li>Server: Windows Server, version 20H2; Windows Server, version 2004; Windows Server, version 1909; Windows Server, version 1809; Windows Server 2019; Windows Server, version 1803</li></ul><div></div><div><strong>Resolution:</strong>&nbsp;This issue was resolved in the out-of-band update&nbsp;<a href='https://support.microsoft.com/help/5001566' target='_blank'> KB5001566</a>. It is a cumulative update, so you do not need to apply any previous update before installing it. You will need to <strong>Check for updates </strong>to install <a href='https://support.microsoft.com/help/5001566' target='_blank'> KB5001566</a> and might need to also select <strong>Download and Install</strong>. For instructions on how to <strong>Check for updates</strong>, please see <a href="https://support.microsoft.com/windows/update-windows-10-3c5ae7fc-9fb6-9af1-1984-b5e0412c556a" rel="noopener noreferrer" target="_blank">Update Windows 10</a>. You can import this update&nbsp;into Windows Server Update Services (WSUS) manually. See the&nbsp;<a href="https://docs.microsoft.com/windows-server/administration/windows-server-update-services/manage/wsus-and-the-catalog-site#the-microsoft-update-catalog-site" rel="noopener noreferrer" target="_blank">Microsoft Update Catalog</a>&nbsp;for instructions.</div><div align='right'><a href ='#known-issues'>Back to top</a></div></td></tr>
</table>

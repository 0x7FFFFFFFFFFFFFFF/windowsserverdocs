<tr><td width='100%' colspan='3'><div>After installing <a href='https://support.microsoft.com/help/4586781' target='_blank'> KB4586781</a> on domain controllers (DCs) and read-only domain controllers (RODCs) in your environment, you might encounter Kerberos&nbsp;authentication issues. This is caused by an issue in how <a href="https://msrc.microsoft.com/update-guide/en-US/vulnerability/CVE-2020-17049" rel="noopener noreferrer" target="_blank">CVE-2020-17049</a> was addressed in these updates. As noted in <a href="https://msrc.microsoft.com/update-guide/en-US/vulnerability/CVE-2020-17049" rel="noopener noreferrer" target="_blank">CVE-2020-17049</a>, there are three registry setting values for PerformTicketSignature to control it, but in the current implementation you might encounter different issues with each setting:</div><ul><li>Setting the value to 0 might cause authentication issues when using S4U scenarios, such as scheduled tasks, clustering, and services for example line-of-business applications.</li><li>The default value setting of 1 might cause non-Windows clients authenticating to Windows Domains using Kerberos to experience authentication issues.</li><ul><li>With setting 1, clients attempting to renew a Kerberos&nbsp;ticket that should be renewable on a DC updated with <a href='https://support.microsoft.com/help/4586781' target='_blank'> KB4586781</a> will fail to renew the Kerberos&nbsp;ticket if it was issued from a DC that has not installed an update released November 11, 2020 or any DC running Windows Server 2008 R2 SP1 or Windows Server 2008 SP2.</li><li>Going from 0 to 1 might also cause this issue since there can be outstanding Kerberos tickets that are marked renewable, but will not be renewed by updated DCs.</li></ul><li>With the default value setting of 1, you might also have <a href="https://docs.microsoft.com/openspecs/windows_protocols/ms-kile/bac4dc69-352d-416c-a9f4-730b81ababb3" rel="noopener noreferrer" target="_blank">Cross realm referrals</a> failures on Windows and non-Windows devices for Kerberos referral tickets passing through domain DCs that have not installed an update released November 11, 2020 or any DC running Windows Server 2008 R2 SP1 or Windows Server 2008 SP2.&nbsp;This issue might happen if domain environment is partially updated or contains at least one Windows Server 2008 R2 SP1 or Windows Server 2008 SP2.</li><li>Setting the value to 2 is intended for enforcement mode and will create issues in an environment where not all DCs are updated because it will explicitly reject certain types of non-compliant Kerberos&nbsp;tickets. It should also not be used at this time if your environment contains DCs running Windows Server 2008 R2 SP1 or Windows Server 2008 SP2.</li></ul><div></div><div><strong>Note </strong>This issue only affects Windows Servers, Windows 10 devices and applications in enterprise environments.</div><div><br></div><div><strong>Affected platforms:</strong></div><ul><li>Server: Windows Server, version 20H2; Windows Server, version 2004; Windows Server, version 1909; Windows Server, version 1903; Windows Server, version 1809; Windows Server 2019; Windows Server 2016; Windows Server 2012 R2; Windows Server 2012</li></ul><div></div><div><strong>Resolution:</strong> This issue was resolved in the out-of-band update <a href='https://support.microsoft.com/help/4594440' target='_blank'> KB4594440</a>. It is a cumulative update, so you do not need to apply any previous update before installing it. To get the standalone package for <a href='https://support.microsoft.com/help/4594440' target='_blank'> KB4594440</a>, search for it in the&nbsp;<a href="http://www.catalog.update.microsoft.com/home.aspx" rel="noopener noreferrer" target="_blank">Microsoft Update Catalog</a>. You can import this update&nbsp;into Windows Server Update Services (WSUS) manually. See the&nbsp;<a href="https://docs.microsoft.com/en-us/windows-server/administration/windows-server-update-services/manage/wsus-and-the-catalog-site#the-microsoft-update-catalog-site" rel="noopener noreferrer" target="_blank">Microsoft Update Catalog</a>&nbsp;for instructions. <strong>Note </strong><a href='https://support.microsoft.com/help/4594440' target='_blank'> KB4594440</a> is not available&nbsp;from Windows Update and will not install automatically.</div><div align='right'><a href ='#resolved-issues'>Back to top</a></div></td></tr>
